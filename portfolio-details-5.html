Go

I developed a Go game in Java as part of a university project. The game was developed using the MVC design pattern and the Swing GUI toolkit. The game features a graphical user interface, an undo/redo system, a scoring system and a timer. I used the following design patterns:

  • Model-View-Controller: The MVC design pattern was used to separate the game's data (model), user interface (view) and logic (controller). This separation of concerns allowed for a more modular and maintainable design.
  • Command: The command design pattern was used to implement the undo/redo system. This pattern allowed for the encapsulation of actions performed by the user, which could then be undone or redone.
  • Observer: The observer design pattern was used to implement the scoring system. This pattern allowed for the separation of the scoring system's logic from the game's logic.
